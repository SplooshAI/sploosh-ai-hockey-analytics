My Python NHL shot chart app has the following structure:

/src/main.py - The main Python FastAPI application
/src/test_main.py - Tests for the main Python FastAPI application

/src/lib/nhl/nhl_shot_chart.py - My original shot chart module immediately deprecated once the NHL switched to using their new NHL Edge API on 2023.11.07. RIP https://statsapi.web.nhl.com/api/v1 ðŸª¦

/src/lib/nhl_edge - My custom module for working with the NHL Edge API
/src/lib/nhl_edge/main.py - Used to import custom services like the required functions from services.data_loader that /src/main.py references
/src/lib/nhl_edge/json - Sample NHL Edge API responses saved as local JSON for faster development, iteration, and verification of known scenarios or edge cases
/src/lib/nhl_edge/services/data_loader.py - Contains functions to load data from the NHL Edge API
/src/lib/nhl_edge/services/test_data_loader.py - Tests for the data_loader service
/src/lib/nhl_edge/services/utils/fetch.py - Utility module to return an HTTP response as JSON
/src/lib/nhl_edge/services/utils/file_operations.py - Utility module to save JSON data to a local file
/src/lib/nhl_edge/services/utils/test_fetch.py - Tests for the fetch utility module
/src/lib/nhl_edge/services/utils/test_file_operations.py - Tests for the file_operations utility module

/src/lib/qrcode/qrcode_generator.py - My custom module for generating QR codes and returning them as images or HTML
/src/lib/qrcode/test_qrcode_generator.py - Tests for my custom QR code module
/src/lib/qrcode/test_generate_base64_image_exception.py - Additional tests for my custom QR code module
/src/lib/qrcode/test_generate_base64_image_with_matplotlib_figure.py - Additional tests for my custom QR code module

/src/middleware/queryparameters/logger.py - Custom middleware for FastAPI to log the route and query parameters for the route
/src/middleware/queryparameters/test_logger.py - Tests for the logger FastAPI middleware

/src/static - A directory containing images and other assets
